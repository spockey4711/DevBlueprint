DevBlueprint intake builder

Fill in the form to generate a .devblueprint-intake.yml and the exact devblueprint command that applies it - including the target path and, for a monorepo, one variant per package. No account, no server, nothing is sent anywhere: this page runs entirely in your browser.

Project answers

A single-stack project picks one variant; a monorepo scaffolds one repo with many packages, each its own variant. Drives --variant vs repeated --package.

Where to scaffold - the --target path. Not a key in the file (it says where, not what); it goes on the command line below.

Human-facing name; fills {{PROJECT_NAME}}. Optional - defaults to the target directory's basename.

Required. The stack DevBlueprint scaffolds. Matches a devblueprint list slug.

Optional add-on overlays layered onto the variant (--flavor). Mutually exclusive with a monorepo.

Each package is a --package name:variant. The repo shares one set of docs and branches; every package carries its own stack.

Always-deployable branch. Default: master.

Where feature work integrates. Default: develop. Set it equal to the main branch to select the single-branch (trunk) flow.

Which instruction files to emit (--agents). claude is always included.

Inherit a shared baseline's answers (extends). A bare name is resolved from the baseline dirs; a path (with / or a .yml suffix) is resolved relative to the file. Optional.

Opt into SECURITY.md + CODE_OF_CONDUCT.md. Default: off.

Reporting address stamped into the community-health files. Only meaningful when the files are enabled.

Intended deploy target. Recorded and echoed; the runbook is generated by a later wave. Optional - leave blank to omit the key.
