+ git workflow for parallel AI agents

Run parallel Claude agents without collisions.

Point several AI coding sessions at one repo at once - each isolated in its own git worktree, each integrating through a pull request. One command scaffolds the whole workflow: two long-lived branches, a quality gate, conventions, and a CLAUDE.md that teaches the agent the rules.

// what lands in your repo

A professional workflow, from commit one.

Extracted from a real production codebase and made stack-agnostic. The output is plain files you own and edit - not a framework you depend on.

A - isolation

Parallel by construction

One worktree per task, branched off develop. Several agent chats run at once and never touch each other's branch, so sessions never collide.

B - quality gate

A gate that travels with you

lint, typecheck, test, build - wired to your stack and enforced on every commit and in CI. One clear definition of done.

C - conventions

Rules the agent already knows

A CLAUDE.md and an engineering-standards mindset that teach the assistant the workflow up front - keeping a solo repo at team quality.

D - no lock-in

Files you own, no runtime

Everything is copied into your project. There is no runtime and nothing hosted - delete DevBlueprint afterward and nothing breaks.

Install.

All three ship the whole kit, so devblueprint runs without a clone.

npx no install · needs Node
$npx devblueprint list
Homebrew macOS / Linux
$brew install spockey4711/devblueprint/devblueprint
curl | sh installs into ~/.devblueprint, drops a command on your PATH
$curl -fsSL https://raw.githubusercontent.com/spockey4711/DevBlueprint/master/install.sh | sh

Then scaffold a project: devblueprint init --target ~/Projects/myapp --name myapp --variant web-nextjs. The installer honours DEVBLUEPRINT_PREFIX, DEVBLUEPRINT_BIN and DEVBLUEPRINT_VERSION if you want to override the defaults.
