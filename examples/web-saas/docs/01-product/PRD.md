# PRD: web-saas

## Problem

Small B2B founders collect trial signups, sales notes, and activation signals across forms, spreadsheets, and chat threads. They lose the story of which accounts need attention and which product actions actually move users toward activation.

## Goal

Build a focused SaaS workspace that helps a founder review new accounts, qualify leads, and track the first activation milestone without introducing a full CRM.

## Users

- Solo founder selling an early SaaS product.
- Small customer-success team reviewing trial accounts.

## MVP Scope

- Account list with status, owner, and activation state.
- Account detail page with notes and latest activity.
- Manual qualification workflow.
- Basic dashboard showing activated, at-risk, and unreviewed accounts.

## Non-Goals

- Billing.
- Email automation.
- Multi-workspace permissions.
- Full CRM pipeline management.

## Acceptance Criteria

- A user can create and update an account.
- A user can mark an account as qualified or not qualified.
- The dashboard reflects account status changes.
- Tests cover the core workflow from account creation to qualification.
